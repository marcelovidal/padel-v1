## BUG 1 — RLS bloquea UPDATE a clubs

El log muestra:
clubs UPDATE result: [] null

El UPDATE tiene clubId válido pero devuelve 
array vacío — RLS está bloqueando la escritura.

El admin (userId: e2573f7a) no tiene política 
RLS que le permita hacer UPDATE en clubs.

Verificar en Supabase ejecutando:
SELECT policyname, cmd, qual 
FROM pg_policies 
WHERE tablename = 'clubs';

El fix es usar el cliente de Supabase con 
service_role key en lugar del cliente normal 
para este UPDATE específico, o crear una 
política RLS que permita al admin actualizar clubs.

En lib/actions/club-owner.actions.ts:
Importar el cliente admin/service de Supabase:
import { createClient } from '@supabase/supabase-js'
const supabaseAdmin = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL!,
  process.env.SUPABASE_SERVICE_ROLE_KEY!
)

Usar supabaseAdmin para el UPDATE a clubs:
const { data, error: clubError } = await supabaseAdmin
  .from('clubs')
  .update({...})
  .eq('id', clubId)
